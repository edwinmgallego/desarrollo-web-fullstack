Cómo hacer un git pull

El comando git pull se utiliza para actualizar tu repositorio local con los últimos cambios del repositorio remoto. Esto es útil cuando necesitas asegurarte de que tu copia local está al día con los cambios realizados por otros colaboradores en el proyecto.

¿Qué hace git pull?

git pull es en realidad una combinación de dos comandos de Git:

git fetch

Descarga los cambios del repositorio remoto, pero no los aplica inmediatamente a tu copia local

git merge

Fusiona los cambios descargados en tu rama actual.

Con git pull, ambos pasos se realizan automáticamente en una sola línea de comando.

Cómo usar git pull

Primero, abre la línea de comandos. Puedes hacerlo en Visual Studio Code (VS Code) dentro de la carpeta de tu proyecto. Usa el atajo "Ctrl + Ñ" para abrir la consola o terminal. Asegúrate de estar ubicado dentro de la carpeta de tu repositorio antes de continuar.

Luego, en la terminal, ejecuta el comando git pull de la siguiente forma: "git pull origin nombre-de-la-rama"

origin:

es el nombre predeterminado del repositorio remoto, osea del origen de donde quieres traer los archivos o lineas de codigo.

nombre-de-la-rama:

se refiere a la rama desde la cual deseas obtener información para actualizar tu copia local. Generalmente, esta es la rama main o master.

En la siguiente imagen, podemos ver un ejemplo en el que la rama se llama "camp-jhon-escarraga". Realizaremos un pull sobre esta rama para descargar su información y aplicarla en nuestra copia local. También es posible permanecer en esta rama y ejecutar el comando "git pull origin main", lo cual traerá los archivos y la información de la rama "main" hacia nuestra rama "camp-jhon-escarraga".

Posibles resultados:

  1. Sin conflictos: Si no hay conflictos, Git fusionará los cambios automáticamente.

  2. Con conflictos: Si hay conflictos entre tu copia local y los cambios del repositorio remoto, Git te indicará cuáles archivos tienen conflictos para que los resuelvas manualmente.

Recuerda realizar un git pull frecuentemente para mantener tu copia local al día y evitar conflictos más complicados en el futuro.
